## Operating UX Readiness Rule

Before first-production validation, GPAO-T must prove:

- Permission and execution states are legible to a non-developer Korean user.
- Preview, review, approval, execution, audit, rollback, and blocked states are visually distinct.
- Model output cannot visually or technically become tool/CLI/MCP execution without proposal, approval, audit, and replay boundaries.
- Connector, MCP, CLI, external send, paid action, and destructive action have visible authority tiers.
- CLI, Work Surface, Control Center, app shell, and future desktop shell share the same authority language.
- Automation and self-growth candidates appear first as reviewable queues, not hidden active processes.
