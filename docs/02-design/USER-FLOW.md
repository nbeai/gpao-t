# User Flow

Map the first visible entry point, first workflow, result, and next action before implementation.

## First Visible Entry

The user opens the GPAO-T core work surface and sees a draft task input area rather than a dashboard-only status wall.

## First Workflow

1. User reads or edits the draft task request.
2. GPAO-T shows a preview thread and current task state without submitting anything.
3. User opens native no-script readability details for the task brief, context/skill route, and authority boundary.
4. GPAO-T previews Context Mesh / Memory Wiki / T-cell candidates for the request.
5. GPAO-T previews Skill Pack routing and model/tool route intent.
6. GPAO-T shows authority/approval status and the closed execution boundary.
7. GPAO-T gives the next safe action.

## Current Result

The first slice is read-only and no-script. It now includes native details/summary readability sections so the user can inspect the task brief, context/skill route, and authority boundary without submitting anything. It proves the work surface shape before live chat, external model connectors, tool execution, connector activation, durable memory promotion, self-growth apply, deployment, messenger, or automation exist.

## Next Action

Run desktop/mobile visual QA for the richer `/work-surface` readability interaction, then decide the next user-visible read-only refinement. Live execution remains blocked.
