# Owner Ops First Scenarios

Status: v0.1 local baseline  
Scope: No-API workflows

## Scenario 1. 음식점 리뷰 답변

입력:

```text
음식은 맛있었는데 대기 시간이 너무 길었어요.
사장님이 친절하고 커피가 정말 좋았습니다.
위생이 조금 걱정됐어요. 다음에는 개선됐으면 좋겠습니다.
```

기대 결과:

- 긍정/부정/혼합 리뷰 분류
- 긴급 확인 필요 리뷰 표시
- 답변 초안 생성
- 자동 게시 차단
- 로컬 기록과 개선 후보 생성

## Scenario 2. 쇼핑몰 문의 분류

입력:

```text
배송 언제 출발하나요?
사이즈가 안 맞으면 교환 가능한가요?
블랙 색상 재입고 언제 되나요?
```

기대 결과:

- 배송/교환환불/재입고/상품 정보 분류
- 사장님 확인 필요 항목 표시
- 답변 초안 생성
- 환불/취소/외부 발송 차단

## Scenario 3. 예약 문의 초안

입력:

```text
이번 주 토요일 오후 3시에 네일 예약 가능한가요? 가격도 궁금해요.
```

기대 결과:

- 예약 가능성 확인 전 상태 표시
- 부족한 질문 표시
- 고객 응대 초안 생성
- 예약 확정/캘린더 쓰기/결제 차단

## 공통 완료 기준

```text
입력 -> 분류/요약 -> 답변 초안 -> 확인 필요 항목 -> 잠긴 행동 -> 로컬 기록 -> 리플레이 요약
```
