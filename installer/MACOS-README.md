# nBeAI. GPAO-T macOS 설치 파일

이 폴더는 macOS에서 GPAO-T를 설치하는 일반 사용자용 패키지입니다.

## 설치

1. 압축을 풉니다.
2. `GPAO-T-Install.command`를 더블클릭합니다.
3. 설치가 끝나면 브라우저에서 GPAO-T 대시보드가 열립니다.

Node.js, Docker, Git을 따로 설치할 필요가 없습니다. 설치 파일에 GPAO-T가
사용할 실행 환경이 포함되어 있습니다.

처음 설치하는 Mac에서도 설정 파일과 로컬 전용 연결 정보가 자동으로
생성됩니다. 사용자가 토큰이나 비밀번호를 입력하거나 복사할 필요가 없습니다.
연결 정보는 해당 Mac의 `~/.gpao-t` 안에만 저장되고, 대시보드는 설치기가
자동으로 엽니다.
설치기가 처음 여는 브라우저 연결은 로컬 GPAO-T 실행 환경에서 자동 승인되므로
사용자가 `gpao-t devices approve ...` 명령을 직접 실행할 필요가 없습니다.

모델 연결 정보가 아직 없는 새 Mac에서도 설치 자체는 완료됩니다. 이 경우
첫 실행 화면에서 모델 연결 설정을 이어가면 됩니다. 모델 연결 전 상태는
설치 실패가 아니라 사용 전 설정이 필요한 상태입니다.

설치 위치는 `~/.gpao-t`이며, 기존 호환 런타임은 기본적으로 읽거나 변경하지
않습니다. 따라서 기존 설치와 함께 사용할 수 있습니다.
설치된 GPAO-T는 GitHub Releases의 GPAO-T 전용 업데이트 feed를 기본으로
기억합니다. 새 버전은 기존 `~/.gpao-t` 사용자 데이터와 설정을 보존하면서
검증된 배포 파일만 단계적으로 적용하는 구조를 사용합니다.

## 설치 후

- 대시보드: `http://127.0.0.1:18799/chat?session=main`
- 모델 연결 설정: `http://127.0.0.1:18799/model-connection.html`
- 상태 확인: `~/.gpao-t/current`와 GPAO-T 백그라운드 서비스
- 사용자 데이터: `~/.gpao-t`

설치 과정은 배포 파일의 SHA-256 무결성을 확인하고, 실패 시 현재 상태를
보호하기 위한 스냅샷과 설치 기록을 남깁니다.

설치 과정에서 런타임이 정상적으로 시작되지 않으면 최근 런타임 로그와
서비스 상태를 함께 표시합니다. 예전 설치에서 보이던 `Missing config` 오류는
새 설치에서 자동 설정 생성 단계로 처리됩니다.
`모델 연결 설정 필요` 상태는 런타임이 정상 설치된 뒤 표시되는 첫 실행
안내입니다.
API 키 또는 provider 연결이 필요한 경우 먼저 `모델 연결 설정` 화면을 열고,
연결 상태를 확인한 뒤 새 대화에서 실제 모델 응답이 나오는지 확인합니다.
API 키와 토큰 값은 화면이나 로그에 다시 표시하지 않는 방향으로 다룹니다.
`장치 페어링 필요` 화면이 보이면 설치기가 자동 승인과 재연결을 처리해야
합니다. 같은 화면이 계속 남아 있으면 브라우저 새로고침 후 같은 주소로
다시 들어가면 됩니다.

## 문제 복구

설치가 실패해도 설치 파일은 삭제되지 않습니다. 터미널의 오류 내용을
확인한 뒤 같은 설치 파일을 다시 실행할 수 있습니다. 이미 설치된 버전의
상태와 롤백은 개발자용 설치 명령으로 확인할 수 있습니다.

이 패키지는 Apple Silicon과 Intel Mac을 함께 지원하는 macOS용 유니버설
패키지입니다. Windows용 설치기는 같은 GPAO-T 배포 계약을 사용하되 Windows
서비스와 경로 규칙에 맞춘 별도 패키지로 제공합니다.
